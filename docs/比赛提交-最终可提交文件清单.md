# NutriMind 计算机设计大赛最终可提交文件清单

本文档用于指导团队整理“源代码”和“代表性素材”提交包，尽量满足以下目标：

- 保留团队自主开发的全部核心源码
- 排除开发工具、编译产物、缓存、日志和超大环境文件
- 将总体积控制在 1GB 以内，最好控制在 500MB 以内
- 保证评委可依据文档重新部署并生成一致的安装包

## 一、建议提交的主文件结构

建议在仓库外新建一个比赛提交目录，例如：

```text
作品编号-提交材料/
├─ 作品编号-安装包.apk
├─ 作品编号-素材源码.zip
└─ 备用说明/
```

其中：

- `作品编号-安装包.apk`：最终 Android 安装包
- `作品编号-素材源码.zip`：源码与代表性素材压缩包

## 二、源码包内建议保留的内容

建议将以下文件和目录整理到 `作品编号-素材源码.zip` 中的 `源码/` 目录。

### 1. 前端与管理端源码

- `frontend-app/`
- `admin-web/`

说明：

- `frontend-app` 为 uni-app 移动端源码，是比赛核心展示端。
- `admin-web` 为后台管理系统源码，虽然不是移动端主赛点，但可体现系统完整性和可维护性，建议保留。

### 2. 后端微服务源码

- `gateway-service/`
- `user-service/`
- `food-service/`
- `meal-service/`
- `ai-service/`
- `common-core/`

说明：

- 这些目录构成作品完整业务逻辑。
- `common-core` 为公共模块，虽不是独立服务，但属于团队开发代码的一部分，应一并提交。

### 3. 构建与部署脚本

- `scripts/`
- `docker/`
- `.mvn/`
- `pom.xml`
- `mvnw`
- `mvnw.cmd`
- `docker-compose.dev.yml`

说明：

- 这些文件用于本地部署、容器初始化、统一构建和启动流程。
- 比赛要求“根据文档能产生完全一样的安装包”，这些构建与部署文件建议保留。

### 4. 配置模板与说明文档

- `.env.example`
- `README.md`
- `docs/TEAM_ONBOARDING.md`
- `docs/图像识别模块协作说明.md`
- `docs/图像识别训练基础说明.md`
- `docs/食物图像识别模块队友运行说明-完整路径版.md`
- `docs/比赛提交-最终可提交文件清单.md`
- `docs/比赛提交-readme模板.txt`

说明：

- 只保留说明性质文档，不提交真实敏感配置。
- `.env.example` 保留变量名和示例值，方便评委理解部署过程。

### 5. AI 相关轻量配置文件

建议保留：

- `ai-service/model/retrieval_bank.json`
- `ai-service/model/category_manifest.json`
- `ai-service/model/clip_text_bank.npz`
- `ai-service/model/clip_text_bank.meta.json`
- `ai-service/model/labels.json`
- `ai-service/model/metadata.template.json`
- `ai-service/model/ai4food_subcategory/README.md`
- `ai-service/model/ai4food_subcategory/metadata.template.json`
- `model/ai4food_subcategory/metadata.json`
- `model/ai4food_subcategory/subcategory_classes.txt`

说明：

- 这些属于模型配置、标签、概念库、检索库和运行说明，可体现团队在图像识别和知识检索上的工作。
- 文件体积很小，建议保留。

## 三、明确不建议提交的内容

以下内容不建议放入源码包，否则容易超体积，且不符合“源代码”范围。

### 1. 版本控制和 IDE 文件

- `.git/`
- `.idea/`

原因：

- `.git` 体积非常大，且不属于比赛要求的源码内容。
- `.idea` 是开发工具配置，不属于作品源码。

### 2. 编译产物和打包缓存

- 所有服务目录下的 `target/`
- `frontend-app/dist/`
- `frontend-app/unpackage/`
- `frontend-app/.vite/`
- `frontend-app/.hbuilderx/`
- 根目录 `dist/`

原因：

- 这些均为构建产物或本地缓存，不属于原始源码。

### 3. 日志与临时文件

- `.run-logs/`
- `hs_err_pid*.log`
- `replay_pid*.log`
- 所有 `__pycache__/`

原因：

- 这些文件不参与项目构建，且无提交必要。

### 4. 本地环境与虚拟环境

- `.ai4food-safe-env/`
- `ai-service/inference/python/.venv/`
- 其他本地 Python/Node/Java 环境目录

原因：

- 这些目录体积很大，且属于本地依赖环境，不是作品源码。

### 5. 真实敏感配置

- `.env`
- `.env.local`
- 各类真实密钥、数据库密码、第三方 API Key

原因：

- 比赛提交材料中不应包含敏感信息。
- 只提交 `.env.example` 即可。

### 6. 超大模型权重文件

当前项目中最重要的超大文件为：

- `model/ai4food_subcategory/efficientnetv2_subcategory_model.hdf5`

该文件大小约为 1.35GB，不建议直接放入源码包。

建议处理方式：

- 在源码包中保留模型调用代码、推理脚本、标签文件、模型说明文件
- 在安装说明中写明模型文件名称、放置路径、版本和获取方式
- 如学校允许，可在说明中提供单独下载方式或答辩现场部署版本

## 四、代表性素材建议提交内容

建议单独整理到 `作品编号-素材源码.zip` 中的 `代表性素材/` 目录。

### 1. 建议提交的代表性素材

- APP 图标、Logo、启动页图片
- 首页、识别页、顾问页、社区页等关键页面展示图
- 用于演示图像识别的典型测试图片 3 到 5 张
- 后台管理系统首页或关键页面截图 1 到 3 张
- 作品宣传图、海报、片头图等自制视觉素材
- 若答辩视频中使用了自制音频或视频片段，可保留代表性文件

### 2. 不建议提交的素材

- 大批量重复测试图片
- 全量训练数据集
- 大量网络下载但与作品展示无直接关系的素材
- 无法说明来源的大量背景音乐或视频素材

### 3. 素材包建议结构

```text
代表性素材/
├─ logo/
├─ app页面截图/
├─ 图像识别测试图/
├─ 后台页面截图/
└─ 素材说明.txt
```

## 六、你这个项目当前最推荐的实际提交策略

结合项目现状，建议按如下方式提交：

### 1. 源代码包

提交内容：

- 全部团队自主开发源码
- 部署脚本
- 配置模板
- 模型轻量配置文件
- 图像识别与检索说明文档
- 代表性素材

不要提交：

- Git 历史
- IDE 配置
- 编译产物
- 本地虚拟环境
- 大模型权重
- 真正的密钥和密码

### 2. 安装包

提交内容：

- 最终 Android APK

建议命名：

- `作品编号-安装包.apk`

### 3. 超大模型文件处理

如果评审系统严格限制体积，则建议：

- 不在源码包中放入 `.hdf5` 模型权重
- 在 `readme.txt` 中说明模型文件路径与获取方式
- 答辩现场使用已部署好的服务器环境进行演示

## 七、最终操作清单

在正式压缩前，请逐项检查：

- 删除或排除 `.git/`
- 删除或排除 `.idea/`
- 删除或排除所有 `target/`
- 删除或排除 `frontend-app/unpackage/`
- 删除或排除 `frontend-app/dist/`
- 删除或排除 `.ai4food-safe-env/`
- 删除或排除 `ai-service/inference/python/.venv/`
- 删除或排除 `.hdf5` 大模型文件
- 删除或排除 `.env`
- 保留 `.env.example`
- 保留部署说明和模型说明
- 保留最终 APK
- 保留代表性素材

如果按上述方式整理，你的源码包体积通常会远低于 1GB，满足比赛提交要求。
