## 二、队友只需要配这次模块的哪些内容

队友只需要处理 3 件事：

1. 模型放对位置
2. 在根目录 `.env` 里补两个变量
3. 给这个模块准备 Python 环境并启动

其他数据库、Docker、JDK、原项目配置，继续沿用他们自己原来的环境即可。

## 三、第一步：确认模型文件放对位置

队友需要确认下面这个目录存在：

```text
项目根目录\model\ai4food_subcategory
```

如果按你这台电脑的完整路径写，就是：

```text
D:\安装网上应用集\Git\NutriMind_1\model\ai4food_subcategory
```

里面要有这 3 个文件：

```text
efficientnetv2_subcategory_model.hdf5
metadata.json
subcategory_classes.txt
```

## 四、第二步：在根目录 `.env` 里补两个配置

队友需要改的文件是：

```text
D:\安装网上应用集\Git\NutriMind_1\.env
```

如果队友电脑项目根目录不同，就改成：

```text
队友自己的项目根目录\.env
```

只需要在里面补这两个值：

```env
APP_VISION_ENGINE=python
APP_VISION_PYTHON_BASE_URL=http://localhost:8091
```

说明：

- `APP_VISION_ENGINE=python`
  表示让 Java 侧使用 Python 推理服务
- `APP_VISION_PYTHON_BASE_URL=http://localhost:8091`
  表示 Python 推理服务默认跑在本机 `8091`

除了这两个值，其他数据库密码、JWT、端口、Nacos 这些继续用队友原来的配置即可。

## 五、第三步：创建这个模块自己的 Python 环境

进入这个目录：

```text
D:\安装网上应用集\Git\NutriMind_1\ai-service\inference\python
```

如果队友项目根目录不同，就进入：

```text
队友自己的项目根目录\ai-service\inference\python
```

推荐直接运行这个脚本：

```powershell
powershell -ExecutionPolicy Bypass -File .\create-ai4food-safe-env.ps1
```

这个脚本的完整路径是：

```text
D:\安装网上应用集\Git\NutriMind_1\ai-service\inference\python\create-ai4food-safe-env.ps1
```

如果队友的 `conda.exe` 不在默认位置，就这样运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\create-ai4food-safe-env.ps1 -CondaExe "队友自己的conda.exe完整路径"
```

执行成功后，项目根目录会多出一个新目录：

```text
D:\安装网上应用集\Git\NutriMind_1\.ai4food-safe-env
```

这个目录是这个模块专用的 Python 环境。

## 六、第四步：启动 Python 推理服务

还是在这个目录下：

```text
D:\安装网上应用集\Git\NutriMind_1\ai-service\inference\python

运行：

```

这个启动脚本的完整路径是：

```text
D:\安装网上应用集\Git\NutriMind_1\ai-service\inference\python\start-ai4food-subcategory.ps1
```

这个 Python 可执行文件的完整路径是：

```text
D:\安装网上应用集\Git\NutriMind_1\.ai4food-safe-env\python.exe
```

启动成功后，这个 PowerShell 窗口不要关。

## 七、第五步：检查 Python 推理服务有没有起来

另开一个 PowerShell 窗口，执行：

```powershell
Invoke-RestMethod http://127.0.0.1:8091/health
```

正常应该看到这些关键信息：

- `classifier_available = true`
- `classifier_runtime = tensorflow`
- `active_backend = tensorflow_retrieval`

如果这一步不通，就不要继续往下排 Java。

## 八、第六步：只重启 `ai-service`

因为队友其他模块已经正常，所以这里只需要让 `ai-service` 重新读取这次新增的两个环境变量。

也就是说：

- 如果 `ai-service` 原来没开，直接正常启动它
- 如果 `ai-service` 原来开着，先停掉，再重启一次

在启动 `ai-service` 的那个终端里，先执行：

```powershell
$env:APP_VISION_ENGINE='python'
$env:APP_VISION_PYTHON_BASE_URL='http://localhost:8091'
```

然后在项目根目录执行：

```powershell
.\mvnw.cmd -pl ai-service spring-boot:run
```

`mvnw.cmd` 的完整路径是：

```text
D:\安装网上应用集\Git\NutriMind_1\mvnw.cmd
```

## 九、第七步：怎么验证这个功能

验证分两层。

### 1. 先看 Python 模型服务

执行：

```powershell
Invoke-RestMethod http://127.0.0.1:8091/health
```

确认 Python 模型服务已经正常。

### 2. 再看项目里的识别功能

前端进入“智能识别”页面，上传一张食物图片。

正常流程应该是：

1. 图片上传成功
2. 系统返回若干个候选食物
3. 用户选择最像的那个
4. 系统再去匹配原有食物库和营养信息

也就是说，这个模块不是直接“看图就返回最终营养值”，而是：

```text
图片 -> AI4Food 候选 -> 食物库匹配 -> 营养信息返回
```
